v0.2.1 (2026-04-03)
--------------------
  - op_tstate: T-stavy od zacatku instrukce (pro contended memory timing)
  - RD/WR/IO_RD/IO_WR makra s op_tstate trackingem
  - M1_FETCH (4T, m1_state=1) vs FETCH (3T, m1_state=0)
  - wait_cycles se nyni pricitaji v NEXT makru
  - z80_add_wait_states() pricita i do op_tstate
  - Q registr pro spravne F3/F5 u SCF/CCF (Patrik Rak, 2018)
  - EI callback (z80_set_ei)
  - 410 testu (8 novych pro Q registr)
  - ZEXALL validace: 67/67 PASS

v0.2 (2026-04-02)
------------------
Prvni vydani multi-instance verze Z80A emulatoru.

Instrukcni sada:
  - Kompletni instrukcni sada Z80A vcetne nedokumentovanych instrukci
  - CB, ED, DD/FD, DD CB/FD CB prefix instrukce
  - Nedokumentovane: IXH/IXL/IYH/IYL, SLL (CB 30-37), DD CB kopie
  - Presne T-stavy, flagy (vcetne F3/F5), MEMPTR/WZ
  - IM0/1/2, NMI, EI delay, LD A,I/R HW bug
  - DAA lookup tabulka (2048 zaznamu)

API:
  - z80_create() / z80_destroy() - dynamicka alokace, callbacky s user_data
  - z80_step(), z80_execute() - batch execution
  - z80_int(), z80_irq(), z80_nmi()
  - z80_get_reg() / z80_set_reg() / z80_is_halted()
  - z80_add_wait_states()
  - Dynamicke settery: z80_set_mread(), z80_set_pwrite() atd.
  - z80_set_post_step(), z80_set_intack(), z80_set_reti()

Optimalizace:
  - Computed goto dispatch
  - Lokalni registrova cache v z80_execute()
  - Lokalni cache callback pointeru (eliminuje overhead multi-instance)
  - FORCE_O2 na z80_execute() (stabilni vykon i pri -O3)

Testovaci sada:
  - 402 testu - vsechny PASS
  - ZEXALL validace: 67/67 PASS
