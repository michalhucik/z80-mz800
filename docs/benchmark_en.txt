==============================================================================
  Z80 Emulator Benchmark: z80ex vs cpu-z80 family
==============================================================================

  Platform:   Windows 10/11, MSYS2/MinGW64, GCC
  Test:       16,777,216 loop iterations, mixed instruction workload
              (arithmetic, logic, memory, CB prefix, DD prefix,
              PUSH/POP, conditional jump)
  Cycles:     2,214,609,436 T-states (identical across all emulators)
  Tests:      402/402 PASS for all cpu-z80 variants

==============================================================================
  BACKGROUND
==============================================================================

  The MZ-800 emulator used z80ex 1.1.21 as its CPU core. To achieve
  better performance and accuracy, we developed a custom Z80 emulator
  (cpu-z80) with a fundamentally different architecture:

  z80ex:
    - Generated opcode handlers from .dat files (Perl script)
    - Each opcode = separate function (function pointer dispatch)
    - ~13,500 lines of code (mostly generated)
    - Only z80ex_step() (single instruction) - no batch execute

  cpu-z80:
    - Hand-written emulator with computed goto dispatch
    - Local register cache in execute loop
    - ~2,000 lines of code
    - z80_execute() with batch execution (main performance difference)

  z80ex-mz800 is our modified z80ex version with a timing bugfix
  (T_WAIT_UNTIL(7) for 16-bit fetch) and EI callback. Benchmarks
  show both z80ex versions are performance-equivalent.

==============================================================================
  VARIANTS
==============================================================================

  z80ex 1.1.21        - Original z80ex (sourceforge.net/projects/z80ex)
  z80ex-mz800         - Our modified z80ex (timing bugfix + EI callback)
  cpu-z80 v0.1        - Our core, single-instance, legacy API
  cpu-z80 multi-v0.2  - Multi-instance, new API (z80ex compatible)
  cpu-z80 single-v0.2 - Single-instance, new API (drop-in with multi)

==============================================================================
  ANALYSIS
==============================================================================

  All cpu-z80 variants are approximately 3x faster than z80ex.

  Main sources of speedup:
  1. Computed goto dispatch vs function pointer dispatch (z80ex)
  2. Batch execution z80_execute() vs per-step z80ex_step()
  3. Local register cache (registers in C locals, not in struct)
  4. Inline prefix handlers (no function calls for CB/ED/DD/FD)
  5. Default callbacks eliminate null-checks in hot path

  z80ex-official vs z80ex-mz800:
  - Performance-equivalent (within measurement noise)
  - Timing bugfix does not change total T-state count, only distribution

  cpu-z80 multi-v0.2 vs single-v0.2:
  - On -O2 equivalent (~3100 MHz, zero multi-instance overhead)
  - On -O3 single is ~6% faster (FORCE_O2 stabilizes performance)

  Note: z80ex has no batch execute, so each instruction requires
  z80ex_step() call -> function pointer dispatch -> return. cpu-z80
  uses z80_execute() with internal computed goto, eliminating
  function call overhead per opcode.
